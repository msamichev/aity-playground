# Ретро: init-project (aity-playground)

> Процессный ретро о самой инициализации. Дата: 2026-06-05.

## 1. Что сделали

Инициализировали полигон `aity-playground` из семени team+ai v0.8.0: наполнили `docs/idea/`
(5) и `docs/product/` (3) из `inbox/idea.md`, заполнили `AGENTS.md` §0, проставили CODEOWNERS,
создали bootstrap ADR. Стек ещё не выбран (Phase 0) — дальше `/adopt-stack`.

## 2. Что прошло хорошо

- Идея (todo-list REST API, in-memory) — простая и однозначная, наполнение docs шло быстро.
- Семя оказалось в чистом Phase 0 (плейсхолдер `<PROJECT_NAME>`, шаблонные docs) — копия легла без правок.
- Решение наполнять только тело файлов, не трогая секции `## Связки`, бережёт граф ссылок (validate-links).

## 3. Что было неочевидно / шероховато

- Семя **ships** placeholder-CODEOWNERS, а Шаг 0 playbook'а трактует наличие CODEOWNERS как
  «уже инициализирован». Распознали placeholder-состояние и продолжили вручную (скилл — обёртка над playbook).
- Выбран GitHub вместо GitLab — методология заточена под GitLab, поэтому часть инфраструктуры
  (glab, GitLab CI, `/open-mr`) потребует адаптации в следующих фазах.

## 4. Что меняем / follow-up

- [ ] `/adopt-stack` → Node+TypeScript (backend-only).
- [ ] Реализовать baseline todo-API + тесты, добиться зелёного `scripts/ci-push.sh`.
- [ ] GitHub: создать репо, защитить `main`, первый push (Maintainer).
- [ ] Позже: адаптация PR-флоу под GitHub (gh, PR-шаблон, GitHub Actions для серверного CI).

## 5. Заметка на будущее

Процедуру init вели не как скилл (он в `.claude/` полигона, не в текущей сессии), а как
playbook вручную — результат идентичен. Для реальных проектов скилл сработает авто-триггером в своей сессии.
