# Check Links Playbook

> **Универсальная процедура.** Читается любым AI: Claude, Codex, DeepSeek, Minimax, Cursor, Gemini, Aider и т.п.
> **Триггер:** `/check-links` или соответствующая фраза на естественном языке (см. таблицу триггеров в [AGENTS.md](../AGENTS.md)).
> **Тонкая обёртка для Claude Code:** [.claude/skills/check-links/SKILL.md](../.claude/skills/check-links/SKILL.md).

---

Запускает валидатор графа знаний с подробным отчётом.

## Алгоритм

1. Запусти валидатор:
   ```bash
   scripts/validate-links.sh --report
   ```
2. Разбери вывод:
   - **Битые ссылки** → перечисли с указанием файла и куда ссылка должна была вести.
   - **Файлы без секции `## Связки`** → перечисли.
   - **Orphan'ы** (warning) → перечисли, спроси нужны ли вообще.
   - **Статистика** → покажи total / errors / warnings.

3. Если ошибки есть — предложи варианты:
   - (a) Починить автоматически (что возможно — отсутствующая секция `## Связки`, легко добавить заглушку).
   - (b) Переименовать перемещённый файл через `scripts/rename-doc.sh OLD NEW`.
   - (c) Удалить ссылку, если файл больше не нужен.

4. После фиксов — перезапусти валидатор.

## Когда запускать вручную

- Перед `/save-all` (push).
- После большой реорганизации `docs/`.
- После переименований через `scripts/rename-doc.sh` (для проверки).
- Периодически: например, раз в неделю в рамках `/week-plan` итога.

## Автоматический запуск

Валидатор и так запускается:
- **На pre-commit** в режиме `--changed` (быстрая проверка изменённых файлов).
- **На CI** в режиме `--full`.

`/check-links` — для случаев когда нужен **подробный отчёт со статистикой**, не просто pass/fail.

## Что считается ошибкой (fail)

- Ссылка на несуществующий файл/анкор.
- Md-файл в `docs/architecture/`, `docs/idea/`, `docs/adr/`, `docs/product/` без секции `## Связки` (whitelist: README/INDEX/template).

## Что считается warning (НЕ fail)

- Orphan: файл, на который никто не ссылается.

Orphan'ы бывают валидными — например, шаблон `template.md` или новый ADR, который только что создан. Поэтому это warning, разработчик решает сам.
