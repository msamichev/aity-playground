# Experiments Audit Playbook

> **Универсальная процедура.** Читается любым AI: Claude, Codex, DeepSeek, Minimax, Cursor, Gemini, Aider и т.п.
> **Триггер:** `/experiments-audit` или фразы «аудит экспериментов», «почисти experiments», «что в experiments».
> **Тонкая обёртка для Claude Code:** [.claude/skills/experiments-audit/SKILL.md](../.claude/skills/experiments-audit/SKILL.md).

---

Сканирует `experiments/` на кандидатов к удалению. **Ничего не удаляет автоматически** — печатает список с обоснованием, пользователь сам решает что делать.

Назначение: в команде с активным AI `experiments/` быстро превращается в свалку одноразовых прототипов. Без регулярного аудита через 3-6 месяцев папка становится непроходимой; ссылки на эксперименты в ADR/планах указывают на устаревший контент. Раз в месяц проходим аудит, удаляем мёртвое отдельным `chore/experiments-cleanup` MR.

## Когда использовать

- Регулярно — раз в месяц (например, в день общекомандной ретро после релиза).
- Перед major-релизом — чтобы не тянуть протухший контент.
- Если `experiments/` визуально распух (>20 папок или ≥100MB).

## Алгоритм

### Шаг 1. Сбор папок experiments/

```bash
find experiments -mindepth 1 -maxdepth 1 -type d -printf '%P\n' 2>/dev/null \
  | sort
```

(GNU find; на macOS — `find experiments -mindepth 1 -maxdepth 1 -type d | sed 's|^experiments/||' | sort`.)

### Шаг 2. Для каждой папки — собрать признаки

| Признак | Команда | Интерпретация |
|---|---|---|
| Возраст последнего коммита | `git log -1 --format=%ar -- experiments/<name>` | >2 недели = кандидат |
| Есть ли ссылки из `docs/adr/` или `plans/` | `grep -rl "experiments/<name>" docs/adr/ plans/ 2>/dev/null` | Нет ссылок = кандидат |
| Размер папки | `du -sh experiments/<name>` | >50MB = подозрительный (мог попасть бинарь) |
| Содержимое README (если есть) | `head -3 experiments/<name>/README.md` | Помогает понять что это |

### Шаг 3. Классификация

| Класс | Критерии | Рекомендация |
|---|---|---|
| **Безопасно удалить** | возраст ≥ 2 недель **И** нет ссылок из `docs/adr/`/`plans/` **И** автор подтверждает | удалить |
| **Подумать** | возраст ≥ 2 недель **И** нет ссылок, но автор не подтверждает | спросить автора (через `git log --format='%an <%ae>' -- experiments/<name>` найди автора) |
| **Оставить** | возраст < 2 недель, **ИЛИ** есть ссылки из ADR/планов | оставить |
| **Подозрительный размер** | размер > 50MB | проверить руками: не попал ли бинарь, не нужно ли вытащить в Git LFS |

### Шаг 4. Печать отчёта

Формат отчёта:

```
=== Аудит experiments/ — 2026-05-22 ===

Всего папок: 14

🟢 Оставить (5):
  - 2026-04-12-prompt-eval-loop   (5 дней назад, ссылается из docs/adr/20260413-1500-eval-strategy.md)
  - 2026-05-18-pytest-bench       (вчера)
  - ...

🟡 Подумать (6):
  - 2026-01-03-langchain-poc      (4 месяца, нет ссылок) — автор: alice
  - 2026-02-15-claude-streaming   (3 месяца, нет ссылок) — автор: bob
  - ...

🔴 Безопасно удалить (3) — после подтверждения авторов:
  - 2025-11-08-old-prompt-template (6 месяцев, нет ссылок, README пуст)
  - 2025-12-20-deprecated-flow     (5 месяцев, нет ссылок)
  - 2026-01-01-test-spike          (4 месяца, нет ссылок)

⚠️ Подозрительный размер (1):
  - 2026-03-22-model-fine-tune    (180 MB) — проверь: бинарь или нужен Git LFS?

Действия:
  1. Подтверди у авторов (🟡, 🔴): «удаляем?». Можно списком в Slack/чате.
  2. Создай ветку: chore/experiments-cleanup-2026-05
  3. git rm -r experiments/<подтверждённые>
  4. /save-all → /open-mr → ревью → merge
```

### Шаг 5. Не удалять автоматически

`/experiments-audit` **только печатает отчёт**. Удаление — ручной шаг через отдельный MR с CODEOWNERS approve, чтобы случайно не сломать ссылки из ADR.

## Правила

- **Не удалять самому.** Только печатать отчёт.
- **Не считать «нет коммитов» окончательным аргументом** — некоторые эксперименты лежат как референс. Всегда смотри ссылки из `docs/adr/` и `plans/`.
- **Не классифицировать как «удалить» без подтверждения автора** — папка может быть нужна автору для будущего возврата.

## Что НЕ делает

- **Не удаляет файлы.** Только отчёт.
- **Не создаёт MR.** Это делает пользователь после подтверждений.
- **Не проверяет `inbox/`** — он по другому правилу (gitignored `inbox/private/`, остальное — снимок входных данных).
- **Не трогает `prompts/`** — там стабильные шаблоны, не одноразовые эксперименты.
