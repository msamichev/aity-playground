# 03. Principles

> **Frozen snapshot.** Заполняется командой `/init-project` из inbox/ и обсуждений.
> После заполнения — не редактируем спонтанно. Изменения — через ADR.

> Принципы: что важно делать так, а не иначе. Что мы НЕ делаем.

## Содержание

**Делаем так:**
- in-memory хранилище, никакой БД/файлов/внешних сервисов;
- чистый JSON REST, корректные HTTP-коды (201 создано, 200 ок, 204 удалено, 400 валидация, 404 нет);
- честная минимальная валидация (пустой `title` → 400; неизвестный `id` → 404);
- тесты на каждую фичу — это ground-truth для оценки ботов;
- маленькие узкие задачи, одна область на ветку.

**НЕ делаем (вне скоупа):**
- аутентификацию, пользователей, роли;
- персистентность, миграции, кэш;
- фронтенд;
- сложную бизнес-логику (приоритеты, дедлайны, теги) — можно добавить позже узкой задачей,
  но не в baseline.
