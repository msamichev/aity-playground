# 05. Risks

> **Frozen snapshot.** Заполняется командой `/init-project` из inbox/ и обсуждений.
> После заполнения — не редактируем спонтанно. Изменения — через ADR.

> Что может пойти не так. По каждому риску — митигация или признак «пока не знаем».

## Содержание

| Риск | Митигация |
|---|---|
| Слабая модель бота не дотянет даже узкую задачу | задачи крошечные; гейты ловят брак; мейнтейнер — финальный гейт |
| In-memory стейт «течёт» между тестами | сбрасывать стор между кейсами; стор инкапсулирован, а не глобальный синглтон без reset |
| GitHub вместо GitLab → часть методологии (glab/CI) не из коробки | адаптируем `/open-mr` на `gh`/PR позже; серверный CI — минимальный GitHub Actions к Фазам 5–6 |
| Полигон «разрастётся» в настоящий проект | скоуп заморожен в `03-principles.md`; рост только узкими задачами |
| Параллельные боты конфликтуют | непересекающиеся области (разные эндпоинты/файлы), «один файл = одна запись» |
