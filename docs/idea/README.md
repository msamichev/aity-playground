# docs/idea/

**Frozen snapshot** идейного ядра проекта на момент инициализации.

## Принцип

- Это **снимок**, не живой документ. Заполняется один раз (на `/init-project`), потом редактируется крайне редко.
- Если мысль эволюционирует — пишется ADR. Соответствующий файл `idea/` может быть обновлён только как часть осознанного действия.
- `idea/` отдельно от `architecture/`. Идея — про «что и зачем», архитектура — про «как».

## Что здесь, а что в других папках

В `docs/idea/` живёт **только идейное ядро** — то, что нужно AI-агенту как контекст:

| Тема | Где живёт |
|---|---|
| Идейное ядро (что/зачем, ресёрч, принципы, MVP, риски) | `docs/idea/` (этот каталог) |
| Короткие продуктовые формулировки (vision/audience/goals) | [`docs/product/`](../product/) |
| Высокоуровневая архитектура | [`docs/architecture/overview.md`](../architecture/overview.md) |
| Этапы развития (MVP → Pilot → Scale roadmap) | [`docs/architecture/roadmap/`](../architecture/roadmap/) |
| Архитектурные и значимые продуктовые решения | [`docs/adr/`](../adr/) |
