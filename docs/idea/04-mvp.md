# 04. MVP

> **Frozen snapshot.** Заполняется командой `/init-project` из inbox/ и обсуждений.
> После заполнения — не редактируем спонтанно. Изменения — через ADR.

> Минимум для показа: что обязательно (capabilities — что система должна уметь), что отложено.
> Если применимо — этапы развития: MVP → Pilot → Scale.

## Содержание

**Capabilities (обязательно в baseline):**
- `POST /todos` — создать задачу `{title}` → `201` с `{id, title, done:false, createdAt}`;
- `GET /todos` — список всех задач → `200`;
- `GET /todos/:id` — одна задача → `200` или `404`;
- `DELETE /todos/:id` — удалить → `204` или `404`;
- `PATCH /todos/:id` — переключить/задать `done` → `200` или `404`;
- `GET /health` — liveness → `200 {status:"ok"}`.

**Обязательно сопутствующее:**
- юнит/интеграционные тесты на каждый эндпоинт;
- `scripts/ci-push.sh` зелёный (build, типы, тесты, покрытие);
- запускаемый сервис (`npm start` / dev-скрипт).

**Отложено (кандидаты в задачи ботам):**
- фильтры/пагинация списка, поля `tags`/`dueDate`, обновление `title`.
