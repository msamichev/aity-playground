# Audience

> Кто пользователь, его профиль, главные jobs-to-be-done.

**Пользователь:** мейнтейнер репозитория (человек) и боты-агенты «aity», работающие по
методологии team+ai. Реальных конечных пользователей у todo-API нет — это полигон.

**Профиль:** один разработчик-мейнтейнер + N дешёвых LLM-ботов в контейнерах.

**Jobs-to-be-done:**
- (мейнтейнер) дать боту узкую задачу и получить проверяемый результат (ветка/PR);
  отличить «сделано» от «не сделано» по фактам гейтов, а не по словам бота.
- (бот) взять задачу, реализовать по методологии, прогнать гейты, отчитаться.
- (оба) иметь стандартный, однозначный API, где у каждой фичи есть явный ожидаемый ответ.
