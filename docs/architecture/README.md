# docs/architecture/

**Живая** архитектура проекта. В отличие от `docs/idea/` (frozen snapshot) — расширяется через ADR.

## Структура

| Папка | Что внутри |
|---|---|
| `core/` | Ядро системы: главный цикл, ключевые компоненты, права, способности |
| `data/` | Сущности, схема связей, нерушимые правила (инварианты данных) |
| `flows/` | Триггеры, пути данных, главные сценарии |
| `integrations/` | Внешние системы (третьи стороны, API, очереди) |
| `stack/` | Технологический стек: подробнее, чем в AGENTS.md §1 |
| `nfr/` | Non-functional requirements: безопасность, расходы, перформанс, риски |
| `roadmap/` | Дорожная карта: первая фича, ближайшие шаги |

## Правила

- Каждый md в этих подпапках обязан иметь секцию `## Связки`.
- При любом изменении — проверка `scripts/validate-links.sh`.
- Решения принимаются через ADR в [`docs/adr/`](../adr/).
