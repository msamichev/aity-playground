# 20260605-2059-bootstrap

## Status

Accepted

## Date

2026-06-05

## Decision Maker(s)

- msamichev

## Context

Проект `aity-playground` инициализирован из семени методологии **team+ai** (версия `0.8.0`, из `VERSION`).
Назначение проекта — **тренировочный полигон** для отладки механики контейнерных ботов «aity»
(инструмент `agent-aity`): задача → feature-ветка → гейты → MR/PR → ревью-мерж.

Источники наполнения `docs/idea/` и `docs/product/`:
- `inbox/idea.md` — набросок идеи (todo-list REST API, in-memory, backend-only);
- обсуждение с мейнтейнером на дату инициализации (стек Node+TS, git-хост GitHub).

Особенность относительно дефолта team+ai: git-хост — **GitHub**, а не GitLab. Это создаёт известный
технический долг (GitLab CI / `glab` / `/open-mr` рассчитаны на GitLab) — разрешается в последующих
фазах плана `agent-aity` (адаптация `/open-mr` на `gh`/PR, минимальный GitHub Actions для серверного CI).

## Decision

Принять методологию из `METHODOLOGY.md` как стандарт работы на этом полигоне.
Стек ещё не выбран на момент bootstrap — это **Phase 0 (Discovery)**; выбор Node+TypeScript
фиксируется отдельно командой `/adopt-stack` (следующий шаг).

## Consequences

### Positive

- Единая структура и контракт (`AGENTS.md`), понятные между сессиями и читаемые любым AI.
- Готовые pre-commit / CI-скрипты на базовом уровне; защита через CODEOWNERS + branch protection.
- Полигон позволяет отлаживать ботов, не рискуя реальным проектом.

### Negative or Trade-offs

- Жёсткая структура методологии ощущается как оверхед для одноразового учебного сервиса.
- GitHub вместо GitLab → часть инфраструктуры методологии не работает «из коробки», нужна адаптация.

### Follow-ups

- `/adopt-stack` — материализовать структуру под Node+TypeScript (backend-only), Phase 1.
- Реализовать baseline todo-API (CRUD + `/health`), добиться зелёного `scripts/ci-push.sh`.
- Создать GitHub-репозиторий, включить branch protection на `main`, первый push (Maintainer).
- Позже: адаптация PR-флоу под GitHub (`gh`, PR-шаблон, GitHub Actions).

## Confirmation

Полигон считается «рабочим», когда на нём проходит хотя бы один автономный прогон бота
(задача → feature-ветка → зелёные гейты), а прямой push в `main` заблокирован branch protection.

## Alternatives considered

### Свободная организация (без методологии)

- Идея: просто накидать todo-API без семени team+ai.
- Отказались потому что: цель полигона — проверить именно методологию + ботов; без неё проверять нечего.

### GitLab как git-хост (дефолт team+ai)

- Идея: использовать GitLab, под который методология заточена (glab, MR, CI).
- Отказались потому что: у мейнтейнера рабочий флоу на GitHub; технический долг адаптации осознанно принят.
