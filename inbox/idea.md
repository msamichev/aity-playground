# Идея: todo-list REST API (полигон)

> Сырьё для `/init-project`. Набросок мыслей, не финальный документ.

## Что это

Маленький REST-сервис для списка задач (todo). Backend-only, без UI, без БД —
хранилище **в памяти** (перезапуск = чистое состояние). Нужен как **тренировочный
полигон**: на нём отлаживаем механику контейнерных ботов «aity» (задача → ветка →
гейты → MR → ревью). Сам сервис намеренно простой и одноразовый.

## Зачем / для кого

Пользователь — мы сами (мейнтейнер + боты). Ценность не в продукте, а в том, что это
**чёткая, тестируемая нагрузка**: у каждой фичи есть однозначный ожидаемый ответ, легко
писать тесты, легко проверить «бот сделал/не сделал». Чем стандартнее API — тем лучше.

## Что должен уметь (capabilities)

- Создать задачу: текст-заголовок → возвращается задача с `id`, `title`, `done=false`, `createdAt`.
- Получить список всех задач.
- Получить одну задачу по `id`.
- Удалить задачу по `id`.
- Отметить задачу выполненной / снять отметку (toggle `done`).
- `GET /health` — liveness-проба (для смоук-теста и будущего CI).

## Принципы / границы (что НЕ делаем)

- Никакой БД, файлов, внешних сервисов — только in-memory структура. Это осознанно.
- Никакой аутентификации, пользователей, многопоточности — не нужно для полигона.
- Никакого фронтенда. Чистый JSON REST.
- Валидация ввода — минимальная, но честная (пустой заголовок → 400; несуществующий id → 404).
- Скоуп заморожен: расширяем только узкими задачами, которые будут давать боты.

## Как поймём, что «работает»

- `npm ci && scripts/ci-push.sh` зелёный локально (типы, тесты, покрытие).
- Эндпоинты отвечают ожидаемыми кодами/телами (есть юнит/интеграционные тесты).
- Запущенный сервис отвечает 200 на `GET /health`.

## Заметки по реализации (не жёстко — решит /adopt-stack)

- Стек уже выбран: **Node + TypeScript** (vitest + eslint).
- Тип проекта: **backend-only** (API).
- Фреймворк (express / fastify / node:http) — на усмотрение `/adopt-stack`; чем легче, тем лучше.
- Git-хост: **GitHub** (не GitLab) — `main` защищён, работа через feature-ветки + PR.
