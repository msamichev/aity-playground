# Миграция team+ai v0.5 → v0.6

Что подтянуть в **уже скопированном проекте** методологии `team+ai`, чтобы получить функциональность v0.6.0 — генератор `.github/copilot-instructions.md` из `AGENTS.md` для GitHub Copilot Workspace.

## Зачем мигрировать

В v0.6.0 закрыт [ADR-1900](https://github.com/msamichev/methodology-ai/blob/main/docs/adr/20260522-1900-gen-copilot-instructions-for-cross-vendor-coverage.md). Cross-vendor coverage расширен до трёх AI:

- **Claude Code** — native через `.claude/skills/` и `AGENTS.md`.
- **Cursor** — через `gen-cursor-rules.sh` (уже в репо).
- **GitHub Copilot Workspace** — теперь через `gen-copilot-instructions.sh` (новый в этой версии).

Для команды это особенно ценно: миксованные конфигурации (часть на Claude Code, часть на Cursor, часть на Copilot Workspace) получают единое содержание из одного `AGENTS.md`, без ручной синхронизации.

**Без миграции 0.5.x продолжает работать.** Миграция нужна, если в команде есть разработчики на GitHub Copilot Workspace.

## Шаги

1. **Скопируй новый скрипт:**

   ```bash
   cp /path/to/team+ai/scripts/gen-copilot-instructions.sh scripts/gen-copilot-instructions.sh
   chmod +x scripts/gen-copilot-instructions.sh
   ```

2. **Запусти генерацию:**

   ```bash
   bash scripts/gen-copilot-instructions.sh
   ```

   Создастся `.github/copilot-instructions.md`. GitHub Copilot Workspace подхватит при следующей сессии.

3. **Реши: коммитить сгенерированный файл или нет.**

   - **Коммитим (рекомендуется для команды):** при изменении `AGENTS.md` release-инженер регенерирует и коммитит. Плюс: все разработчики Copilot работают по актуальной инструкции. Минус: расхождение при отсутствии регенерации.
   - **В `.gitignore`:** каждый разработчик генерирует у себя локально. Минус: рассинхрон между разработчиками.

   Решение зависит от команды. Запиши в `retrospectives/YYYY-MM-DD-release-vX.Y.Z.md` действие «настроить процесс регенерации Copilot-instructions».

4. **Добавь `--check` в `.pre-commit-config.yaml`** (рекомендуется для коммит-формата):

   ```yaml
   - id: gen-copilot-instructions-check
     name: gen-copilot-instructions actuality check
     entry: bash scripts/gen-copilot-instructions.sh --check
     language: system
     pass_filenames: false
     files: ^AGENTS\.md$
   ```

   Это блокирует коммит, если `AGENTS.md` изменён, а `.github/copilot-instructions.md` не регенерирован.

5. **Обнови `playbooks/README.md`** копией из свежего seed — добавится раздел «GitHub Copilot Workspace — рецепт настройки».

6. **Расскажи команде** про новую возможность на ближайшей retro-митинге. Без коммуникации Copilot-пользователи могут не узнать о новой опции.

7. **Обнови `VERSION`** на `0.6.0` руками.

8. **Прогон `scripts/validate-links.sh`** — должен быть зелёным.

## Что НЕ нужно делать

- **Не редактируй `.github/copilot-instructions.md` вручную.** Изменения исчезнут при следующей регенерации. Источник правды — `AGENTS.md`.
- **Не делай регенерацию обязательной для каждого коммита** через невыборочный pre-commit hook. Используй `files: ^AGENTS\.md$` чтобы хук срабатывал только при изменении AGENTS.md.
- **Не настраивай Copilot Workspace для каждого разработчика руками.** Это масштабируется только через генерацию.

## Откат

1. Удалить `scripts/gen-copilot-instructions.sh`.
2. Удалить `.github/copilot-instructions.md` (если был коммичен).
3. Откатить `playbooks/README.md` к v0.5.0 (удалить раздел Copilot Workspace).
4. Если был добавлен pre-commit hook — удалить.
5. Вернуть `VERSION` на `0.5.0`.

Полностью обратимо.
