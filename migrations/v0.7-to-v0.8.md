# Миграция v0.7 → v0.8

Что подтянуть в **уже скопированном проекте** методологии `team+ai`, чтобы получить функциональность v0.8.0 — восстановимость `/autopilot` на длинных прогонах (`--resume`), протокол контрольных точек и опциональные hooks PreCompact/SessionStart.

## Зачем мигрировать

В v0.8.0 закрыт [мета-ADR 20260604-1911](https://github.com/msamichev/methodology-ai/blob/main/docs/adr/20260604-1911-autopilot-context-resumability-and-checkpoints.md): на длинных прогонах autopilot упирался в авто-компактификацию контекста (лоссовую). Теперь состояние — в файлах, прогон **восстановим** (`/autopilot --resume`) в той же feature-ветке, есть контрольные точки со сбросом контекста по масштабу, и два hook-а как страховка от внепланового компакта.

**Без миграции 0.7.x продолжает работать.** Миграция нужна, только если хочешь resume/checkpoints/hooks. Предпосылка — у тебя уже есть `/autopilot` (миграция [v0.6→v0.7](v0.6-to-v0.7.md)). Применяй через MR (как все правки методологии).

## Шаги

1. **Скопируй новые/обновлённые файлы** из свежего seed:

   ```bash
   cp /path/to/team+ai/scripts/autopilot-precompact.sh     scripts/
   cp /path/to/team+ai/scripts/autopilot-session-start.sh  scripts/
   cp /path/to/team+ai/scripts/autopilot-preflight.sh      scripts/   # обновлён: пишет checkpoint
   chmod +x scripts/autopilot-precompact.sh scripts/autopilot-session-start.sh scripts/autopilot-preflight.sh
   cp /path/to/team+ai/playbooks/autopilot.md              playbooks/
   ```

2. **Обнови `plans/TEMPLATE.md`** — в `## Журнал автономного прогона` добавь строку **resume-note** (контрольная точка) из seed.

3. **Обнови `.claude/skills/autopilot/SKILL.md`** (resume-триггеры в `description`) — копией из seed.

4. **Обнови `AGENTS.md`:** строку `/autopilot` в §6 (триггер «продолжи план» + «восстановим») и абзац про resumability/hooks в §6.1 — из seed. `CLAUDE.md` — симлинк, не трогаем.

5. **`.gitignore`** — добавь:

   ```
   .claude/autopilot-checkpoint.md
   ```

6. **Зарегистрируй hooks** в `.claude/settings.local.json` (личный файл, в `.gitignore`; НЕ `settings.json` — его перетирает `gen-claude-deny.sh`). К существующему Stop-хуку добавь:

   ```json
   {
     "hooks": {
       "Stop": [
         { "matcher": "", "hooks": [ { "type": "command", "command": "scripts/stop-checklist.sh" } ] }
       ],
       "PreCompact": [
         { "matcher": "", "hooks": [ { "type": "command", "command": "scripts/autopilot-precompact.sh" } ] }
       ],
       "SessionStart": [
         { "matcher": "", "hooks": [ { "type": "command", "command": "scripts/autopilot-session-start.sh" } ] }
       ]
     }
   }
   ```

7. **Обнови `VERSION`** на `0.8.0`.

8. **Проверки:** `scripts/validate-links.sh` зелёный; `shellcheck scripts/autopilot-*.sh` чисто.

9. **Внеси через MR** (отдельная ветка `chore/<ID>-autopilot-resumability`, `changelogs/unreleased/<ID>-…md`, approve от `@architects`).

## Как пользоваться

- Длинный прогон в своей feature-ветке: на знаковой точке autopilot коммитит фазу (с `Refs:`) и пишет resume-note — и **продолжает сам**, не прерываясь ради сброса контекста (тот ужимается автоматически: auto-compact + субагенты). Останавливается только на настоящих блокерах.
- Свежий контекст — **по твоему желанию** (необязательно): сам сделал `/clear` → SessionStart hook напомнит продолжить, `/autopilot --resume` восстановит состояние и продолжит с первой невыполненной фазы (в той же feature-ветке).
- Checkpoint `.claude/autopilot-checkpoint.md` создаётся pre-flight и удаляется на чистом финале. Push/MR остаются ручными (`/open-mr`).

## Что НЕ нужно делать

- **Не клади hooks в `.claude/settings.json`** — он автогенерируется. Только `settings.local.json`.
- **Не коммить `.claude/autopilot-checkpoint.md`** (личный, в `.gitignore`).
- **autopilot по-прежнему не пишет в `main`** — resume работает только в твоей feature-ветке.

## Откат

1. Удалить `scripts/autopilot-precompact.sh`, `scripts/autopilot-session-start.sh`.
2. Откатить `scripts/autopilot-preflight.sh`, `playbooks/autopilot.md`, `plans/TEMPLATE.md`, `.claude/skills/autopilot/SKILL.md`, `AGENTS.md`, `.gitignore` к 0.7.0.
3. Убрать PreCompact/SessionStart из `.claude/settings.local.json`.
4. Вернуть `VERSION` на `0.7.0`.

Полностью обратимо.
