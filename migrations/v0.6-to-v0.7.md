# Миграция v0.6 → v0.7

Что подтянуть в **уже скопированном проекте** методологии `team+ai`, чтобы получить функциональность v0.7.0 — команду `/autopilot` (автономное выполнение многофазного плана по фазам в собственной feature-ветке, с коммитом на фазу и без push).

## Зачем мигрировать

В v0.7.0 закрыт [мета-ADR 20260604-0231](https://github.com/msamichev/methodology-ai/blob/main/docs/adr/20260604-0231-autopilot-phased-autonomous-execution.md): добавлен `/autopilot` — исполнитель плана поверх существующих кирпичей (`/critic`, four-gate субагенты, Stop hook, `dangerous-commands.md`). Берёт план целиком, идёт по фазам в твоей feature-ветке, коммитит каждую фазу с `Refs:` (без push), при предметном сомнении сверяется с интернетом, вопросы задаёт только на старте, на тупике останавливается с отчётом. Push в `main` запрещён всегда; открытие MR (`/open-mr`) — ручной шаг.

**Без миграции 0.6.x продолжает работать.** Миграция нужна, только если хочешь команду `/autopilot`. autopilot опирается на инфраструктуру autonomous mode — если она ещё не подтянута, подтяни её сначала (это базовая миграция, синхронная с solo+ai v0.2→v0.3).

## Шаги

1. **Скопируй новые файлы** из свежего seed:

   ```bash
   cp /path/to/team+ai/playbooks/autopilot.md            playbooks/autopilot.md
   mkdir -p .claude/skills/autopilot
   cp /path/to/team+ai/.claude/skills/autopilot/SKILL.md .claude/skills/autopilot/SKILL.md
   cp /path/to/team+ai/scripts/autopilot-preflight.sh     scripts/autopilot-preflight.sh
   chmod +x scripts/autopilot-preflight.sh
   ```

2. **Добавь секцию `## Журнал автономного прогона`** в `plans/TEMPLATE.md` — между `## Открытые вопросы` и `## Итог` (скопируй блок из свежего seed).

3. **Обнови `AGENTS.md`:**
   - В таблице §6 добавь строку `/autopilot`.
   - В §6.1 добавь абзац про autopilot (feature-ветка, без push в `main`, границы с `/autonomous` и `/full-ahead`).
   - `CLAUDE.md` — симлинк на `AGENTS.md`, отдельно править не нужно.

4. **Обнови `playbooks/plan.md`** (Шаг 3 + `## Связки`) и **`playbooks/README.md`** (строка `autopilot.md`) — копией фрагментов из свежего seed.

5. **Проверь готовность авто-режима** (autopilot использует four-gate): `.claude/agents/` содержит `code-reviewer`, `test-runner`, `security-auditor`, **`merge-coordinator`**; `scripts/stop-checklist.sh` исполняемый; Stop hook зарегистрирован в `.claude/settings.local.json`. Проверить: `bash scripts/autopilot-preflight.sh` из своей feature-ветки (предупредит о недостающих компонентах).

6. **Обнови `VERSION`** на `0.7.0` руками.

7. **Прогон `scripts/validate-links.sh`** — должен быть зелёным.

8. **Внеси изменения через MR** (как все правки методологии в team+ai): отдельная ветка `chore/autopilot-migration`, `changelogs/unreleased/<ID>-autopilot.md`, approve от `@architects`.

## Как пользоваться

- Перейди в свою feature-ветку (`git switch -c feature/<ID>-slug`), `git fetch && git rebase origin/main`.
- На готовом плане: `/autopilot` → подтверди allowlist разрешений одним блоком → autopilot идёт по фазам, коммитя каждую с `Refs:` (без push).
- Крупная фича без плана: `/autopilot` предложит развилку **(A)** план вместе / **(B)** полностью автономно.
- После прогона: проверь → `/save-all` (push в feature-ветку) → `/open-mr` (self-review + merge-coordinator gate). Откат — `git reset --hard <базовый SHA>` (печатает pre-flight).

## Что НЕ нужно делать

- **Не запускай autopilot на `main` или чужой ветке** — pre-flight откажет (exit 2). Только своя feature-ветка.
- **Не запускай на грязном дереве** — нужна чистая база для отката.
- **Не ослабляй deny-лист и запрет push в `main`** ради автономности. autopilot работает с жёстким `dangerous-commands.md` и в авто-режиме.
- **Не путай команды:** `/autonomous` включает режим, `/full-ahead` гоняет один цикл и пушит в feature-ветку, `/autopilot` выполняет весь план без push.

## Откат

1. Удалить `playbooks/autopilot.md`, `.claude/skills/autopilot/`, `scripts/autopilot-preflight.sh`.
2. Откатить правки `AGENTS.md` (§6 / §6.1), `playbooks/plan.md`, `playbooks/README.md`, `plans/TEMPLATE.md` к версии 0.6.1.
3. Вернуть `VERSION` на `0.6.1`.
4. `scripts/validate-links.sh` — зелёный.

Полностью обратимо: autopilot ничего не меняет в существующих командах.
