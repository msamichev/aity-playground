# experiments/

Одноразовые прототипы и черновики кода.

## Правила

- Один эксперимент = одна задача = одна папка `YYYY-MM-DD-slug/`.
- **Может умереть в любой момент** — не считается prod-кодом.
- **Не покрывается mutation testing, не считается в coverage.**
- Если эксперимент удался — фича переезжает в основной код через нормальный `/plan`.
- Папка `experiments/` исключена из `validate-links.sh` — связки тут не обязательны.

Эта папка хороша для:
- Быстрых проверок гипотез
- Изучения новой библиотеки
- Сравнения двух подходов

Эта папка плоха для:
- Долгоживущего кода
- Чего-либо, на что ссылаются другие документы
